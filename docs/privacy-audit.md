# 隐私审计清单 — rikka-agent

> 发布前逐项确认。每项标注"通过"或"待修"。

## 1. 数据收集

| 检查点 | 状态 |
|--------|------|
| 应用不收集、发送任何遥测/分析数据 | ✅ 通过 |
| 不含任何第三方 SDK（广告/分析/崩溃上报） | ✅ 通过 |
| 不联网获取远程配置 | ✅ 通过 |
| 唯一的网络行为是用户主动发起的 SSH 连接 | ✅ 通过 |

## 2. 密钥与凭证

| 检查点 | 状态 |
|--------|------|
| SSH 私钥使用 EncryptedFile (AES-256-GCM) 加密存储 | ✅ 通过 |
| Codex API Key 不以明文形式出现在日志/崩溃报告中 | ✅ 通过 |
| 密码/口令在内存中用后不持久化 | ✅ 通过 |
| Known hosts 指纹存储不包含敏感信息 | ✅ 通过 |

## 3. 日志与输出

| 检查点 | 状态 |
|--------|------|
| Release 构建不输出 SSH 命令/会话内容到 Logcat | ✅ 通过 |
| ProGuard 已配置，混淆发布构建 | ✅ 通过 |
| 命令输出保存在本地 Room DB，不上传 | ✅ 通过 |
| 用户可以删除单个会话/全部历史 | ✅ 通过 |

## 4. 网络安全

| 检查点 | 状态 |
|--------|------|
| `network_security_config.xml` 禁止明文 HTTP 流量 | ✅ 通过 |
| SSH 连接使用 sshj 库自身的加密（不依赖 TLS） | ✅ 通过 |
| Host key 验证默认开启（TrustFirstUse） | ✅ 通过 |
| 不存在 SSRF 向量（无 HTTP 客户端，无 URL 打开） | ✅ 通过 |

## 5. 权限

| 检查点 | 状态 |
|--------|------|
| 仅申请 INTERNET 权限 | ✅ 通过 |
| 不申请存储/定位/相机/麦克风等敏感权限 | ✅ 通过 |
| 密钥导入通过 SAF (Storage Access Framework) 选择器 | ✅ 通过 |

## 6. 数据导出控制

| 检查点 | 状态 |
|--------|------|
| 会话导出仅通过 Share Intent（用户主动触发） | ✅ 通过 |
| 导出内容不自动包含密钥/密码 | ✅ 通过 |
| Clipboard 操作仅复制公钥/命令输出（无私钥） | ✅ 通过 |

## 7. 开源合规

| 检查点 | 状态 |
|--------|------|
| 所有依赖许可证兼容 Apache-2.0 | ✅ 通过 |
| 无 GPL/AGPL 强 copyleft 依赖 | ✅ 通过 |
| LICENSE 文件存在 | ✅ 通过 |
| 第三方许可声明页面（AboutScreen）存在 | ✅ 通过 |
